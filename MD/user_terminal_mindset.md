**Elijah's own words (2026-04-19):**
*"I'm completely new to Linux but I'm sold on it. I've never been in a command terminal this much. I no longer use the computer. I use the command terminal — which lets me know I'm not using the computer. I'm programming it."*

**Why this matters:**
- He's a self-taught non-programmer who recently internalized that the terminal is a different *mode* of computer use — not a harder UI, but a more powerful *role*. "User" vs. "programmer" is the frame.
- This is the exact narrative arc a buyer would experience with Master AI: they arrive with a mouse-and-menu mental model, Master AI nudges them into terminal-first thinking, they level up.
- Pupil's curriculum (Linux/bash → Python) was already designed for this arc. Elijah just articulated the *why* in a marketing-ready line.

**How to apply:**
- In docs/marketing/slideshow copy, lean into "programming the computer, not just using it" as the value prop — it captures the mindset shift better than "learn Linux" or "get productive."
- When onboarding a non-technical buyer, frame the first lessons as *"your first step from user to operator."*
- Don't write docs that assume terminal familiarity — Elijah's journey is the customer's journey. He's the profile: non-programmer, sold on the potential, willing to learn because the tool pays off.
- Tailor tone in Sensei/Pupil replies: respectful of the learner, not condescending; assume they can handle real commands but explain the "why" behind each.
