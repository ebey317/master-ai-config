## M9 contract

**`/chat` response shape** (additions on top of M0):
- `turn_id` (uuid per call)
- `turn_root` (chain root — first turn's id)
- `round_num` (1-indexed)
- `round_budget` (default 3, ceiling 8)
- `round_remaining` (budget - round_num)
- `done` (bool — true on `no_actions` | `budget` | `duplicate_failure`)

**`/chat/continue`** (new endpoint): POST `{parent_turn_id, action_results[], prompt?, session_id, source}`. Auth-gated identically to `/chat`. Mints a new `turn_id`, bumps `round_num`, carries `turn_root`. If `prompt` is omitted, synthetic "continue" is used.

**Backend safety:** `_duplicate_failure_reason()` short-circuits when the same `(kind, target)` shows `result:"failure"` twice in one round's `action_results`. Returns synthetic `route:"loop_safety"`, `model:"m9_short_circuit"`, `latency_ms:0`, `done:true`.

**Turn-level audit:** every terminal turn writes one JSONL record to `~/.master_ai_audit_typed.jsonl`:
```
{"kind":"turn_terminal", "turn_id":..., "turn_root":..., "round_num":..., "round_budget":..., "terminal_reason":"no_actions"|"budget"|"duplicate_failure", "route":..., "model":..., "actions_count":..., "blocked_count":...}
```

**Extension state.loop:**
- `active, turn_id, round_num, round_budget, pending, rejected, results[], stopped`
- `recordLoopResult` called from both `approveAction` + `rejectAction` after `reportAction`
- 50ms `setTimeout` defers `continueLoop` to let the audit POST + UI updates flush
- Hard-stop on any reject in a round (loop.rejected=true → continueLoop returns "Loop stopped — at least one action rejected")
- Fresh user prompt always `resetLoop()`s first
